Зміст бізнес-процесів роботи автоматизованого сервісного центру з надання поліцейських послуг

I Характеристика об’єкта автоматизації

  1. Постановка задачі та технічне завдання.
    1. Детальний опис предметного середовища (об’єкта управління), що моделюється.
    2. Мета розробки проекту (кінцеві результати роботи системи).
    3. Бізнес-задачі, що моделюються в системі, для досягнення мети.
    4. Бізнес-процеси, що здійснюються в системі, що моделюється (на основі деталізації задач).
    5. Сценарій дій системи (з т. з. користувача).
    6. Технічне завдання за ГОСТ 34.602-98 – формалізація функціональних та нефункціональних вимог замовника.
  2. Об’єктно-орієнтовний аналіз (ООА) предметного середовища (об’єкту управління) для версії 1 проекту.
    1. Об’єктно-орієнтовна декомпозиція предметного середовища (діаграма об’єктів).
    2. Функціональні вимоги замовника - UML-діаграми (прецедентів (use-cаse), відповідно до стандарту IEEE Std 830-1998 IEEE Recommended Practice for Software Requirements Specifications-Description (для версій, в яких змінюється функціональність)
  3. Об’єктно-орієнтоване проектування (OOD) для усіх версій проекту.
    1. Проектування структури ПЗ - UML-діаграма класів (classes), з обґрунтуванням типів зв’язків між класами, атрибутами та методами (для усіх версій).
    2. Проектування алгоритмів та взаємодії методів - UML-діаграми діяльності (activity - версія4), послідовності (sequence - версія 5).
    3. Проектування станів об’єктів UML-діаграма станів і переходів (ctatechart версія 6) для версії.
    4. Проектування взаємодії об’єктів (communication для версії 6).
  4. Реалізація проектів по версіях.
    1. HIPO діаграма, яка зображує ієрархію викликів функцій та інформаційні потоки даних (можлива генерація звіту програмами створення документації).
    2. Опис програми та згенерована документація по коду проекту(документація на підвищення балу за звіт).
    3. Файлова структура проекту (скрін шот Solution explorer).
    4. Специфікації функцій у вигляді таблиці з переліком прототипів функцій, семантики функцій та їх параметрів (дивись таблицю).
